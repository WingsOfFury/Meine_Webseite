## Wie kam ich auf die Idee ?
<p>
Wie es dazu kam das ich mich für die Kombo [Astro](https://astro.build/) + [TailwindCSS](https://tailwindcss.com/) entschieden habe lag an dem Youtuber [Programmieren mit Mario](https://www.youtube.com/@ProgrammierenMario).
(Am Ende des Blogs verlinke ich dir das Video)
</p>

<p>
Es gab vor einiger Zeit einen Hype von Englischsprachigen und Deutschen Devs auf Youtube die Astro in den höchsten Tönen gelobt haben.
Zu diesem Zeitpunkt war ich auch auf der Suche nach einem neuen Techstack mit dem ich meine eigene Page bauen wollte.
Da ich schon immer entweder mit Bootstrap oder TailwindCSS arbeiten wollte passte das mit TailwindCSS super.
Also entschied ich mich für Astro/TailwindCSS.
</p>

## Was war dir wichtig und was waren No-Gos ?
<p>
Oberste Priorität war es für mich den Code selber zu schreiben und kein Template zu benutzen.
Es gibt unmengen an Top Templates for free oder gegen Cash aber 
sie bringen sehr wenig lerneffekte mit sich da der Code eh schon fertig geschrieben ist.
</p>

## Was hast du unternommen um die Page weiter auszubauen ?
<p>
Nachdem ich nun den Kurs beendet habe und die Page 1:1 wie in dem Video aussieht kam bei mir schnell der Gedanke auf noch mehr daraus zu machen.
Schon beim Aufbau der Page schossen mir automatisch viele neue Ideen durch den Kopf die ich gerne umsetzen wollte.
Also auf zur Astro Doku(mentation) Seite.
An dieser Stelle muss ich sagen das die Doku von Astro super gestaltet, erklärt und gut nachvollziehbar ist.
</p>

## Welche Eigenschaften sollte die Page noch haben ?
<p>
Diese Seite habe ich mit dem Mobile First ansatz aufgebaut.
Responsive ist sie natürlich auch.
Ein tolles Tool ist die Google Extension [Responsive Viewer](https://responsiveviewer.org/).
Ich kann zwar mit meinen Desktop Computer, Tablet und Smartphone gleichzeitig arbeiten und jede änderung gut erkennen aber der
Responsive Viewer hat einige Devices mehr zu bieten.
Ein Back-to-Top button + Javascript funktion habe ich auch implementiert.

```js
// Hier holt er sich die ID "top-top-button"
let toTopButton = document.getElementById("to-top-button");

// Checkt ob der Button existiert
if (toTopButton) {
  // Scroll event, toggle button wird erst sichtbar wenn nach unten gescrollt wird
  window.onscroll = function () {
    if (
      document.body.scrollTop > 500 ||
      document.documentElement.scrollTop > 500
    ) {
      toTopButton.classList.remove("hidden");
    } else {
      toTopButton.classList.add("hidden");
    }
  };

  // Funktion um "smooth" BackToTop zu gelangen
  window.goToTop = function () {
    window.scrollTo({
      top: 0,
      behavior: "smooth",
    });
  };
}
```
</p>
